## Usage

```
GBC [OPTIONS]
```

## Options

`-p, --port <port>`

: The HTTP (and WebSocket) port to listen on. Default is 9001.

`-c, --config <config>`

: The JSON file to use for configuration. If not specified, &gbc will start with a minimal configuration. Configuration can be pushed to &gbc after connecting from &gbr.

`-s, --shared-memory <shared-memory>`

: The default is "gbc_shared_mem", only in advanced scenarios should this be changed.

`-g, --signal <signal>`

: The default is 29 (SIG I/O POSSIBLE), only in advanced scenarios should this be changed.

`-l, --log-level <log-level>`

: Sets logging level (integer). The following log levels (in order of increasing verbosity) are:

    * `0` - FATAL
    * `1` - ERROR
    * `2` - WARNING
    * `3` - INFO
    * `4` - DEBUG
    * `5` - TRACE

`-d, --log-category <log-category>`

: Sets the categories of logging desired (comma separated). The following categories are supported:

    * `WS_LOG` - Websockets log enable
    * `JS_LOG` - JavaScript log enable
    * `JS_MEM_LOG` - JavaScript memory management log enable
    * `ACTIVITY_LOG` - Activity log enable
    * `OTG_LOG` - Trajectory planner log enable
    * `HTTP_LOG` - HTTP server log enable
    * `STARTUP_LOG` - GBC startup log enable
    * `STACK_LOG` - stack log enable
    * `TASK_LOG` - task log enable
    * `STATE_LOG` - state log enable (the state of the connection to GBR and PLC)
    * `ACTIVITY_LOG` - activity log enable
    * `TELEMETRY_LOG` - telemetry log enable
    * `FIELDBUS_LOG` - fieldbus log enable
    * `FRAMES_LOG` - frames log enable
    * `KC_LOG` - kinematics configuration log enable
    * `STREAM_LOG` - stream log enable
    * `MACHINE_LOG` - machine log enable
    * `ALL_LOG` - All logging categories enabled. This creates a lot of logging so should generally not be used
