## Introduction

### `moveParameters: {}`
The moveParameters section of the config allows the definition of sets of move parameters that can be referenced by different types of moves in activities.

Move parameters define things like the velocity of a move,

### `task: {}` & `activity: {}`

Within &gbc, Tasks are an assembly of Activities that are executed in sequence in response to an event occurring.

&gbc can have multiple Tasks defined and tasks can contain multiple Activities.

&gbc will execute a Task based on its current state (E.g. whether it is already running or has finished) and whether the task's Trigger has fired.

The trigger for a task would typically be an event like a IO being set or tasks can be triggered from front-end code.

Tasks allow for JavaScript code to be run is response to events occurring or periodically without using the front-end code (browser based). See <Link to="/docs/gbc/response_times"> here </Link> for more details on the response time of code in different layers of GBC.

Tasks can have trigger to start them running. This could be a digital input or a thresholded analog input.


### `lines: {}`
The lines section of the config allows sets of lines to be defined that can be referenced by activities.

### `arcs: {}`
The arcs section of the config allows sets of arcs to be defined that can be referenced by activities.

### `cartesianPositions: {}`
The cartesianPosition section of the config allows sets of arcs to be defined that can be referenced by activities.


### `triggerOn: {}`
The triggerOn section of the config allows triggers to be defined. These are used to trigger tasks or activities to start - for example on a digital input triggering.

## Example JSON

### `moveParameters: {}`

```json
{
  "moveParameters": {
    "set1": {
      "amaxPercentage": 50,
      "blendTimePercentage" : 60,
      "blendTolerance" : 20,
      "blendType" : 0,
      "jmaxPercentage": 10,
      "limitConfigurationIndex" : 0,
      "toolIndex": 1,
      "vmaxPercentage": 25
    }
  }
}
```
