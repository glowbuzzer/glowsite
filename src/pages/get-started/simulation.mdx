## What is simulation mode?
Simulation is a way to test the &gbt without needing any hardware (motors, IO and so on).

It is also used to speed up the development of front-end components by allowing you to build React machine control applications without having to fire-up specific hardware.

{/*
Testing the &gbt in simulation mode means you can exercise a large part of the software on your desktop PC or Raspberry Pi. No Linux real-time extensions are required.
*/}

We offer two demo front-ends built with &gbrl for you to see what is included out of the box. These will connect to an instance of &gbcl running on your local machine or a Linux machine on your network.

    ## Step 1 - Installing and running &gbc
    First you need to install &gbc (our real-time core). Then you will launch one of the demo apps and connect it to &gbc.

    <InstallGbc/>

    To run GBC using the appropriate configuration, run:

    <PageVariantTabs>
        <Tabs.TabPane tab="Generic" key="generic">
            ```shell
            cd &gbc_version;
            ./GBC --config ../configs/config_generic.json
            ```
        </Tabs.TabPane>
        <Tabs.TabPane tab="Staubli" key="staubli">
            ```shell
            cd &gbc_version;
            ./GBC --config ../configs/config_staubli_tx40.json
            ```
        </Tabs.TabPane>
    </PageVariantTabs>

    You should see output similar to the following:

    ```text
    [INFO ] GBC: **************************************************************************
    [INFO ] GBC: *** Starting GB Control ***
    [INFO ] GBC: **************************************************************************
    [WARN ] GBC: No valid port was specified on the command line. Will use the default [9001]
    [WARN ] GBC: No valid signal was specified on the command line. Will use the default [29]

    ```

    Now &gbc is running, the next step is to launch the sample front-end built using &gbr and connect to &gbc.

    ## Step 2 - Launch the front-end
